Full Stack Applications

The right application may require a "full stack" solution. Powerful and unique applications may be created with the proper front-end and back-end technologies. I would like to share a couple of my full stack applications:

Bin Making App

This application is currently being used by employees at Marker Systems LLC. The application was designed to serve several purposes for the manufacturing company:

  1. Automating time consuming tasks.
  2. Creating an interface between production and management staff.
  3. Providing useful analytics about production.

Since implementation - the application has accomplished each of the goals above and there have been countless benefits from use of the app. Including:

  • Reduced labor commited to tasks such as creating pick lists or consolidation lists.
  • Improved coordination between employees.
  • Improved scheduling through use of production analytics.
  • Reduced inventory through use of included tools.
  • Significantly less production mistakes by reducing human error.
Front End

The app features a high-performance interface which uses React.js to display dynamic content. The use of React Router means there are almost zero page loads while moving throughout the app. The app is also uses Tailwind.css to provide a unique by very professionally styled interface.

Back End

This site uses Django REST API. Django was chosen because of Python's ability to handle the large sets of data required to make the app work. Python also allowed for complex operations to be performed for certain tools in the app.

Additionally, some app features required "real-time" functionality. In these instances websockets were used with the help of Django Channels. The result is that multiple users can interact with the app at the same time.

DJ Set Builder

This app has been a real product of passion! As a music lover and DJ hobbiest - I saw an opporunity to take my DJ sets to the next level. So I created my "DJ Set Builder" which makes creating sets an interactive and visual experience. The app works by following these steps:

  1. The user adds their songs using the form. This includes valuable information such as the mood of the song and how energetic it is. A bubble is created for the song and colored for it's energy.
  2. The user can filter songs based on their parameters (such as vibe, energy, or BPM).
  3. The user drag and drops songs into the tracklist on the right. This is where they begin actually building their DJ Set!
  4. The user can view graphs - displaying the flow throughout the DJ set, so they can visualize what it will do to their crowd.
Front End

This application also uses React.js to create a very dynamic user experience. The interface allows for seemless interaction between page objects and is able to communicate with the back end without any page loads. The app uses Bootstrap to create a familiar but tasteful style.

Back End

The DJ Set Builder is built using the "MERN" stack. With this method Express.js is used for the requests and MongoDB is used as the database. This full JavaScript stack works quickly to support the front end.

Automation Scripts

Countless hours are wasted every day performing menial or repetitive tasks. That's where I come in! Scripting may be a great solution to reduce these pesky tasks or to perform complex operations. Scripts are usually simple and do not have a GUI (graphical user interface). They can be run manually or automatically to suit your needs. Here are a couple of scripts I use regularly:

File Sequencer

The File Sequencer came to be when a photography studio needed an better way to manage their large inventory of image files. Upon uploading photo shoots from a digital camera, the image files had obscure names such as dsc_092821.jpg or dsc_100821.jpg, which had nothing to do with the content of the images and they were hard to sort. So the file sequence was created to:

  1. Be able to quickly rename a large volume of images (or files).
  2. Allow the user to provide their own descriptor of the image contents.
  3. Rename the images in sequence so they can be easily sorted.

Needless to say - the script works great! It works well to handle camera uploads but can be used for many similar purposes too. Since it will work with any file type, the File Sequencer has been used to rename hundreds of PDFs at once or thousands of log files at a time. Read more about it on Github.

Telegram Monitor

The Telegram Monitor became necessary for a customer who was having problems with their Crypto mining process. The issue being faced is that their crypto mining operation would often fail or the machine which was mining would occassionally power off without them knowing. This would lead to missed time mining and loss of money. So the Telegram Monitor was made to:

  1. The script runs on a schedule - as frequently or infrequently as necessary.
  2. When run it will notify the user through a Telegram message if their mining operation failed.
  3. Allow the user to respond to the message - telling the machine to restart the operation.
  4. Notify the user again to inform them when the operation was running again.

The Telegram Monitor works exactly as expected. The monitor notifies the customer when there's a problem and they are able to respond immediately. It has drastically limited the amount of down time in their crypto mining process. You can also see how this application can be applied to many other necessary computer processes.

Process Automation

For over a decade - my technical skills have been used to create or enhance processes in the manufacturing environment. Increased labor costs, greater demand, and more complex procresses are just a few reasons why automation is necessary to compete. Here's just a couple of automation projects:

Completely Automatic Manufacturing

The manufacturing process for Niabraze LLC's fixed diamond wire was very powerful and effective. However, it presented a couple of issues. First - it required 24 hour per day monitoring for changes in product quality. Second - the process required a lot of small manual interventions to keep the product in specification. It was clear that there must be a better way...

So I turned to a Indusoft Web Studio - a powerful SCADA (supervisory control and data acquisition) software solution. We began by having Indusoft read from a SQL database, containing data from the process monitoring system. Once the data was being read by Indusoft we were able to create an alerting system if the product's quality was moving out of specification. This solved problem #1.

But for problem #2 we needed to take things a step further - by actually adjusting process parameters. Instead of just alerting the operator, Indusoft was able to actually make subtle adjustments to keep the product in spec. Some adjustments included changing the process speed, changing the temperature of chemical baths, and metering chemical additives into those baths.

The result? A (literally) lights out manufacturing process. Human intervention was nearly eliminated and the process was able to run 24 hours per day - often without the lights on. Product consistency also improved and operators were immediately alerted to any out of spec product without any additonal testing.

Factory Simulation

Creating a new manufacturing process can be a costly, time consuming process. There's a lot of uncertainty involved and risk associated with every decision. For these reasons, factory simulation can be very beneficial before starting a build. Here are just a few benefits to using simulation:

  1. Eliminating logistical errors.
  2. Time cycle studies.
  3. Program and test PLCs, robots, and more offline.

I have primarily used Visual Components for simulation and the benefits have been amazing. It has made for later implementations to go much smoother than would otherwise be possible. Use of this technique ensures success right from the start with any manufacturing process.
